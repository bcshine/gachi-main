[HTML 기반 가치관 공유 앱 - 기능 설계 지침 (로그인 인증 없음)]

이 앱은 사용자들이 "자신이 중요하게 여기는 가치관"을 등록하고, 다른 사용자들이 공감(좋아요)을 표시할 수 있으며, 댓글과 인기 랭킹을 통해 소통할 수 있도록 구성됩니다. 로그인 인증 없이 동작하도록 설계합니다.

🧩 1) 가치관 입력 및 공유
텍스트 입력 폼을 통해 1개의 가치관 등록 가능 (예: "성실함", "자유로운 삶")

하루에 5 번만 등록 가능 (로컬 스토리지를 활용한 제한 적용) , 6번이상 하려고 할때에는 하루5회 한정입니다. 라는 메시지 알려줄 것. 

입력 시 해시태그 자동 제안 기능 제공 (ex: #성장, #연대, #사랑)

등록된 가치관은 리스트 형태로 실시간 반영

👍 2) 좋아요 투표 기능
각 가치관 카드에 '좋아요' 버튼 존재

사용자는 한 가치관에 한 번만 좋아요 가능 (로컬 스토리지로 중복 방지)

좋아요 수를 기준으로 실시간 인기 가치관 순위 갱신

오늘의 인기 가치관 TOP 10 제공 (정렬 기준: 좋아요 수)

💬 3) 채팅 & 반응 기능
각 가치관마다 댓글 입력 가능 (닉네임 입력 없이 단순 댓글)

이모지 리액션 버튼 제공 (ex: 😊 👍 😢 😮)

댓글은 가치관별 스레드 구조로 표시됨 (원글 + 댓글 리스트)

📈 4) 랭킹 및 추천 피드
전체 가치관을 좋아요 수 기준으로 정렬하여 보여줌 (전체 TOP, 오늘 TOP, 주간 TOP 등 구분 가능)

관심 키워드 기반 추천 피드 (입력된 해시태그 기반 정렬 및 필터 기능 제공)

⚙️ 기술 구현 방향 (로그인 없음)
로컬스토리지 활용:

하루 1회 입력 제한

동일 항목 중복 좋아요 방지

HTML + CSS + JavaScript 로 구현

UI는 단일 페이지 형태로 구성 (SPA 아님)

가치관 데이터는 자바스크립트 배열로 관리 (초기 MVP)

필요 시 이후 단계에서 Firebase 또는 백엔드 연동 고려 가능하지만, 현재 목표는 인증 없이 클라이언트 사이드 기반 MVP입니다.
